# The Revenue Blueprint

A free, complete guide to running complex enterprise deals - the kind where more than one person has to say yes, a technical buyer has to sign off, and a business owner has to feel the outcome.

Most sales advice is personality ("be a natural") or hustle ("send more emails"). This is neither. It's a **system in three layers**: why organisations buy, what you build before you engage, and how you run the cycle that closes. Written from real six-figure enterprise deals, not theory.

> Sales isn't a personality you're born with. It's a system you can learn.

## Who it's for

Founders building go-to-market from scratch, sellers stuck on deals that won't move, and teams trying to turn selling from a personality trait into something repeatable. It's written around enterprise motions (buying committees, business cases, enterprise architecture) but the same three layers apply to any deal that carries real risk.

## A note on the frameworks

The underlying frameworks - Porter's value chain, the APQC Process Classification Framework, MEDDPIC - aren't mine, and I point to each as it comes up. What's mine is the synthesis: the way they connect into one continuous system from "why this business would change" all the way to signed.

## Running the site

This is an [Astro](https://astro.build) + [Starlight](https://starlight.astro.build) site. The guide content lives in `src/content/docs/`, and the deployed version is at [blueprint.davistui.com](https://blueprint.davistui.com).

```bash
npm install     # install dependencies
npm run dev     # local dev server
npm run build   # production build to ./dist
```
